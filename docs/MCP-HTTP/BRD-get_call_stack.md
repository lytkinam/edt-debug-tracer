# BRD — HTTP-метод `get_call_stack`

**Версия:** 1.0  
**Дата:** 2026-06-06  
**Статус:** ✅ Метод существует  
**Связанный SRS:** [SRS-get_call_stack.md](SRS-get_call_stack.md)

---

## 1. Контекст

После получения события SUSPEND оркестратор знает только `threadId`.
Чтобы понять, **где именно** остановилась программа и какие фреймы
активны, нужно собрать стек вызовов.

---

## 2. Потребность оркестратора

Оркестратор использует `get_call_stack` для двух целей:
1. Получить `frameId` верхнего фрейма для последующего `get_variables`.
2. Сформировать поле `callStack` итогового ответа агенту.

Достаточно **одного вызова** с `threadId` из `wait_for_break`.

---

## 3. Место в сценарии

```
[ШАГ 3] wait_for_break  →  threadId
           │
           ▼
[ШАГ 4] get_call_stack(project, threadId)   ← этот метод
           │  frameId верхнего фрейма
           ▼
[ШАГ 5] get_variables
```

---

## 4. Ожидаемое поведение

1. Возвращает фреймы в порядке от верхнего (текущего) к нижнему.
2. Каждый фрейм содержит `frameId` — он обязателен для `get_variables`.
3. Содержит `lineNumber` для верификации, что поток остановился именно на
   нужной строке.
