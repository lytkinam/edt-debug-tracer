# BRD — HTTP-метод `get_variables`

**Версия:** 1.0  
**Дата:** 2026-06-06  
**Статус:** ✅ Метод существует  
**Связанный SRS:** [SRS-get_variables.md](SRS-get_variables.md)

---

## 1. Контекст

Основная полезная нагрузка сценария `breakpoint-metrics` — значения переменных
в момент остановки. Оркестратор получает их через `get_variables`,
зная `threadId` и `frameId` из предыдущих шагов.

---

## 2. Потребность оркестратора

- Если `variables` в запросе не пусто — оркестратор фильтрует полученный
  список по именам и возвращает только запрошенные.
- Если `variables` пусто — возвращаются все переменные фрейма.
- Результат формирует поле `variables` итогового ответа агенту.

---

## 3. Место в сценарии

```
[ШАГ 4] get_call_stack  →  frameId
           │
           ▼
[ШАГ 5] get_variables(project, threadId, frameId)   ← этот метод
           │
           ▼
[ШАГ 6] suspend
```

---

## 4. Ожидаемое поведение

1. Возвращает все переменные фрейма без фильтрации — фильтрацию делает оркестратор.
2. Каждая переменная содержит `name`, `type`, `value`.
3. Если переменная ссылается на объект (`hasChildren: true`) — оркестратор
   в текущей версии берёт строковое `value` как есть, без рекурсии.
