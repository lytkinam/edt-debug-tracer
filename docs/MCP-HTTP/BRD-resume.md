# BRD — HTTP-метод `resume`

**Версия:** 1.0  
**Дата:** 2026-06-06  
**Статус:** ✅ Метод существует  
**Связанный SRS:** [SRS-resume.md](SRS-resume.md)

---

## 1. Контекст

После установки точки останова программа находится в приостановленном
состоянии (SUSPEND). Чтобы она дошла до нужной строки, оркестратор должен
дать команду продолжить выполнение.

`resume` — атомарная команда «отпустить поток».

---

## 2. Потребность оркестратора

Оркестратор вызывает `resume` **один раз** после `set_breakpoint`, чтобы
запустить движение к точке. После этого он немедленно переходит к
`wait_for_break` — ожиданию нового SUSPEND-события.

`resume` не является блокирующим вызовом: HTTP-сток отправляет команду
отладчику и сразу возвращает ответ. Ожидание берёт на себя оркестратор
через `wait_for_break`.

---

## 3. Место в сценарии `breakpoint-metrics`

```
[ШАГ 1] set_breakpoint
           │
           ▼
[ШАГ 2] resume(project, threadId)   ← этот метод
           │
           ▼
[ШАГ 3] wait_for_break
           ...
```

---

## 4. Ожидаемое поведение

1. Команда применяется к конкретному потоку (`threadId` из `debug_status`).
2. Вызов **не блокирует** оркестратора — возвращает ответ немедленно.
3. При повторном `resume` на уже выполняющийся поток — не ошибка,
   либо игнорируется.
