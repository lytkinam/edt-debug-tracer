# BRD — HTTP-метод `set_breakpoint`

**Версия:** 1.0  
**Дата:** 2026-06-06  
**Статус:** ⛔ ТРЕБОВАНИЕ (метод отсутствует в HTTP-стоке)  
**Автор требования:** оркестратор `tracer-sink breakpoint-metrics`  
**Связанный SRS:** [SRS-set_breakpoint.md](SRS-set_breakpoint.md)

---

## 1. Контекст

Оркестратор `breakpoint-metrics` реализует сценарий сбора состояния программы
в заданной точке исходного кода. Первым и обязательным шагом сценария
является установка точки останова в отладчике EDT перед запуском выполнения.

Без этого шага программа пройдёт мимо нужной строки, и оркестратор не
получит событие SUSPEND.

---

## 2. Потребность оркестратора

Оркестратор должен сообщить отладчику:
- в каком файле проекта установить точку;
- на какой строке.

После успешного вызова отладчик EDT будет приостанавливать выполнение при
достижении указанной строки. Оркестратор продолжает сценарий (шаг `resume`).

---

## 3. Место в сценарии `breakpoint-metrics`

```
[ШАГ 1] set_breakpoint(project, file, line)
           │
           ▼
[ШАГ 2] resume(project, threadId)
           │
           ▼
[ШАГ 3] wait_for_break(project, timeoutMs)
           ...
```

---

## 4. Требования к HTTP-стоку

1. Метод **обязан быть реализован** до релиза `breakpoint-metrics`.
2. Метод должен быть **идемпотентным**: повторный вызов с теми же параметрами
   не должен вызывать ошибку.
3. При попытке установить точку на несуществующую строку — вернуть ошибку
   с понятным сообщением (не молча проигнорировать).
4. Метод должен вернуть управление синхронно: точка установлена до `return`.

---

## 5. Ожидаемый ответ для оркестратора

Оркестратору достаточно знать:
- точка установлена (`set: true`);
- идентификатор точки (`breakpointId`) для возможной очистки в будущем.
