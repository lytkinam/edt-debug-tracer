# 05. Интеграция с Vanessa

## Концепция

Vanessa вызывает MCP HTTP API **вокруг каждого шага** BDD-сценария:
- перед шагом → `POST /mcp/start`
- выполняет шаг через TestClient
- после шага → `POST /mcp/stop` + сохраняет трейс

## BSL-хук в шаге Vanessa

```bsl
// В библиотеке шагов Vanessa:
Процедура ПередКаждымШагом(КонтекстШага)
    HTTPСоединение = Новый HTTPСоединение("localhost", 18080);
    Запрос = Новый HTTPЗапрос("/mcp/start");
    HTTPСоединение.ОтправитьДляОбработки(Запрос);
КонецПроцедуры

Процедура ПослеКаждогоШага(КонтекстШага)
    HTTPСоединение = Новый HTTPСоединение("localhost", 18080);
    Запрос = Новый HTTPЗапрос("/mcp/stop");
    Ответ = HTTPСоединение.ОтправитьДляОбработки(Запрос);
    ЗаписатьТрейсВЛог(Ответ.ПолучитьТелоКакСтроку());
КонецПроцедуры
```

## Сохранение трейса в отчёт Allure

Treyс в JSON можно сохранить как Allure attachment:
```bsl
ПрисоединитьФайлКШагу("bsl-trace.json", ТрейсJSON, "application/json");
```

## Важное ограничение

Плагин работает **только когда EDT подключён к отладочному сеансу** (`debug target` запущен и EDT является клиентом отладки). Если сеанс 1С запущен без отладки — `SUSPEND` события не приходят, буфер остаётся пустым.
