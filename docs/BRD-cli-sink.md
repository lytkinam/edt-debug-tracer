# BRD — EDT Debug Tracer: CLI-Сток

**Версия:** 1.0  
**Дата:** 2026-06-06  
**Репозиторий:** lytkinam/edt-debug-tracer, ветка `feature/trace-sink`  
**Статус:** Draft

---

## 1. Бизнес-контекст

Плагин EDT Debug Tracer (`TestActivator`) разворачивает HTTP-сервер внутри Eclipse EDT
и предоставляет полный программный доступ к отладочной сессии 1C:Enterprise через
набор `/mcp/*` и `/sink/*` эндпоинтов. Это инструмент общего назначения: он
намеренно широк, расширяем и никогда не уменьшается в своих возможностях.

Параллельно с ростом инструмента возникает задача встраивания трейсера в конвейеры
автономных ИИ-агентов. ИИ-агент не является конечным пользователем с экраном —
он является **оркестратором**, который вызывает внешние инструменты как чёрные ящики,
ожидая детерминированного, предсказуемого поведения.

Широкий HTTP API с ручным управлением состоянием (`/mcp/start` → `/mcp/run` → `/mcp/stop`)
**не подходит** для прямого вызова агентом: агент может зависнуть в промежуточном
состоянии, получить текстовую ошибку, которую начнёт галлюцинировать, или
потерять контекст между вызовами.

---

## 2. Бизнес-проблема

| Проблема | Следствие |
|----------|-----------|
| Агент вызывает `/mcp/start`, затем `/mcp/run` — между вызовами теряется состояние | Трейс начинается без запущенной сессии → ошибка в непредсказуемой точке |
| HTTP возвращает произвольный JSON с текстом ошибки | Агент пытается распарсить текст → галлюцинации вместо retry |
| Нет единого контракта «что подать → что получить» | Каждый сценарий требует от агента знания деталей API |
| Нет явного сигнала «завершено / ошибка» без разбора тела ответа | Агент не знает, успешен ли запуск, без парсинга JSON |

---

## 3. Бизнес-цель

Создать **CLI-сток** — детерминированный оркестратор конкретных сценариев
использования EDT Debug Tracer для ИИ-агентов.

CLI-сток — это не замена HTTP API и не его упрощение. Это **тонкий коннектор**
поверх существующего `/sink/run` эндпоинта, который:

- Принимает один JSON-контракт через `stdin` или аргумент.
- Валидирует контракт **до** обращения к HTTP.
- Выполняет ровно один сценарий — полный цикл трейса.
- Возвращает результат в `stdout` в виде JSON.
- Сигнализирует об исходе через **детерминированный exit code** (0–4).
- Описывает себя через `--help` и `--version`.

HTTP-сервер при этом **не изменяется** и продолжает расти независимо.

---

## 4. Заинтересованные стороны

| Роль | Интерес |
|------|---------|
| ИИ-агент (LLM-оркестратор) | Вызывает CLI как чёрный ящик; ожидает exit code и JSON в stdout |
| Разработчик плагина | Сохраняет полный HTTP API без изменений; CLI не добавляет сложности в ядро |
| QA / CI-пайплайн | Использует CLI для автоматизированного сбора трейсов в тестах |
| Пользователь командной строки | Может вызывать CLI вручную для разовых сценариев |

---

## 5. Бизнес-требования

**BR-1.** CLI-сток реализует ровно один сценарий: запуск отладочной сессии,
сбор трейса и возврат результата. Никакой другой логики в CLI нет.

**BR-2.** Интерфейс CLI стабилен: входной контракт и exit-коды не меняются без
мажорной версии. HTTP API при этом может расширяться без ограничений.

**BR-3.** CLI не содержит бизнес-логики. Вся логика трейса остаётся в
`TracerSink.java` и `TestActivator.java`. CLI — только валидация контракта,
один HTTP-вызов и маппинг ответа в exit code.

**BR-4.** При любой ошибке агент немедленно знает **тип** проблемы по exit code
без парсинга тела ответа. Тело `stdout` содержит детали для логирования.

**BR-5.** CLI документирует себя: `--help` возвращает машиночитаемое и
человекочитаемое описание контракта; `--version` — текущую версию.

---

## 6. Ограничения и допущения

- HTTP-сервер `TestActivator` запущен и доступен на `localhost:18080`
  (или по адресу из конфигурации) до вызова CLI.
- CLI не управляет жизненным циклом HTTP-сервера.
- CLI является скриптом (Python 3.8+) без дополнительных зависимостей,
  кроме стандартной библиотеки.
- CLI поставляется в репозитории в каталоге `scripts/`.

---

## 7. Критерии успеха

- Агент вызывает CLI одной командой и получает exit 0 + JSON с трейсом.
- При неверном JSON на входе — немедленно exit 2, без обращения к HTTP.
- При недоступном HTTP-сервере — exit 1.
- `--help` возвращает полное описание контракта.
- Ни одна строка бизнес-логики трейса не дублируется в CLI.
